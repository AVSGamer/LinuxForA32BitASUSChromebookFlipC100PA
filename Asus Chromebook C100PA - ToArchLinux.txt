[Returning to my previously tested method of Official Arch Linux Instructions]

Switch to developer mode
Turn off the laptop.
To invoke Recovery mode, you hold down the ESC and Refresh keys and poke the Power button.
At the Recovery screen press Ctrl-D (there's no prompt - you have to know to do it).
Confirm switching to developer mode by pressing enter, and the laptop will reboot and reset the system. This takes about 15-20 minutes.
Note: After enabling developer mode, you will need to press Ctrl-D each time you boot, or wait 30 seconds to continue booting.

Enable booting from external storage
After booting into developer mode, hold Ctrl and Alt and poke the T key. This will open up the crosh shell.
Type shell to get into a bash shell.
Type sudo su to become root.

Type the following to enable booting into unsigned USB OSes:
crossystem dev_boot_usb=1 dev_boot_signed_only=0

Reboot the system to allow the change to take effect.

Create a root USB or SD for dual booting(Official Arch Instructions-Based):
These instructions are written for installing to a USB drive with the sda device, assuming no other USB drives are plugged in. For an SD card, click here to magically adjust the instructions for the mmcblk1 device that an SD card will register as.
Get a root shell as described in the previous section.(sudo su)
Since ChromeOS will automatically mount any partitions it finds, unmount everything now:
umount /dev/sda*
Start fdisk to create a GPT partition table:
fdisk /dev/sda
At the fdisk prompt:
Type g. This will create a new empty GPT partition table.
Exit by typing 'w' key.
Partition the microSD card or USB Drive:
cgpt create /dev/sda
cgpt add -i 1 -t kernel -b 8192 -s 32768 -l Kernel -S 1 -T 5 -P 10 /dev/sda
To create the rootfs partition, we first need to calculate how big to make the partition using information from 'cgpt show'. Look for the number under the start column for Sec GPT table which is 15633375 in this example:
localhost / # cgpt show /dev/sda
       start        size    part  contents
           0           1          PMBR
           1           1          Pri GPT header
        8192       32768      1   Label: "Kernel"
                                  Type: ChromeOS kernel
                                  UUID: E3DA8325-83E1-2C43-BA9D-8B29EFFA5BC4
                                  Attr: priority=10 tries=5 successful=1

    15633375          32          Sec GPT table
    15633407           1          Sec GPT header
Replace the xxxxx(15728607) string in the following command with that number to create the root partition:
cgpt add -i 2 -t data -b 40960 -s `expr xxxxx - 40960` -l Root /dev/sda
Tell the system to refresh what it knows about the disk partitions:
partx -a /dev/sda
Format the root partition:
mkfs.ext4 /dev/sda2
Download and extract rootfs tarball:
cd /tmp
curl -LO http://os.archlinuxarm.org/os/ArchLinuxARM-armv7-chromebook-latest.tar.gz
mkdir root
mount /dev/sda2 root
tar -xf ArchLinuxARM-armv7-chromebook-latest.tar.gz -C root
Flash the kernel to the kernel partition:
dd if=root/boot/vmlinux.kpart of=/dev/sda1
Unmount the root partition:
umount root
sync
Reboot the computer.
At the splash screen, instead of pressing Ctrl-D to go to ChromeOS, press Ctrl-U to boot to the external drive.
After logging in as root (password is "root")

[Volume Key Script] <- make sure that soundcard is working correctly first! The Asus Chromebook Flip C100PA has pretty bad internal speakers audio output on this setup, as long as the headphone jack mode is correctly working let's just leave it at that.
sudo nano /usr/local/bin/vol_change.sh <- '/usr/...' is a globally accessible directory so we put our custom scripts here to be executable/accessible by any account on the system.
[WHAT TO PUT INSIDE THAT vol_change.sh START]
#!/usr/bin/env bash

export XDG_RUNTIME_DIR="/run/user/$(id -u)"

if [ "$1" == "toggle" ]; then
    wpctl set-mute @DEFAULT_AUDIO_SINK@ toggle
else
    # 1. Natively change volume and cap it strictly at a max of 1.00 (100%)
    wpctl set-volume -l 1.0 @DEFAULT_AUDIO_SINK@ "$1"
fi

# 2. Capture the full raw state line to check for mute status
MUTE_STATE=$(wpctl get-volume @DEFAULT_AUDIO_SINK@ 2>/dev/null)

# 3. Safely parse and convert WirePlumber's decimal volume to a clean percentage
VOLUME_RAW=$(echo "$MUTE_STATE" | awk '{print $2}')

# Fallback check if the string comes back completely blank
if [ -z "$VOLUME_RAW" ]; then
    VOLUME=0
else
    # Natively multiply the decimal by 100 using a fast AWK integer tool
    VOLUME=$(echo "$VOLUME_RAW" | awk '{print int($1 * 100 + 0.5)}')
fi

# 4. Interrogate the Physical Kernel Jack State to Toggle Hardware Amplifiers
# This specific model has a physical switch in the Headphone Jack that is handled on the Kernel Level.
# Note: Based on your amixer -c 0 contents results, your jack check(Headphone Jack Status=on|off) utilizes numid=67
HP_JACK_STATE=$(amixer -c 0 cget numid=67 2>/dev/null | grep ": values=" | awk -F= '{print $2}')

if [[ "$HP_JACK_STATE" == "on" ]]; then
    DISPLAY_NAME="Headphones"
else
    DISPLAY_NAME="Speakers"
fi
# Pipewire handles the switching of Audio Routing based on the Jack State
# so no need to explicitly include that operation here. The above is just for what to show in the UI Notification
# as which audio device volume is being changed.

# 5. Check if mute status is active via pattern matching
# If it is muted, it overrides and suppresses volume percentage changes in Dunst
if [[ "$MUTE_STATE" == *"[MUTED]"* ]]; then
    notify-send -a 'Volume' -h string:x-dunst-stack-tag:volume "$DISPLAY_NAME: Muted"
else
    notify-send -a 'Volume' -h string:x-dunst-stack-tag:volume -h int:value:"$VOLUME" "$DISPLAY_NAME: $VOLUME%"
fi
[WHAT TO PUT INSIDE THAT vol_change.sh END]
sudo chown root:root /usr/local/bin/vol_change.sh <- to make root account own it

amixer -c 0 cset name='Right Speaker Mixer Right DAC Switch' on >/dev/null 2>&1

amixer -c 0 cset name='Right Speaker Mixer Left DAC Switch' off >/dev/null 2>&1

# 4. CRITICAL HEADPHONE ROUTING: Maintain working headphone layouts

amixer -c 0 cset name='Right Headphone Mixer Right DAC Switch' on >/dev/null 2>&1
amixer -c 0 cset name='Left Headphone Mixer Right DAC Switch' off >/dev/null 2>&1

[Accidental Power Off via Power Button Protection]
We will make another script that utilizes 'dunst' and it's partner 'dunstify' to make a accidental shutdown via power button prevention notification/confirmation:
sudo nano /usr/local/bin/power_dunst.sh
[WHAT TO PUT INSIDE THAT power_dunst.sh START]
#!/bin/bash

export XDG_RUNTIME_DIR="/run/user/$(id -u)"

# Using the keyword "default" maps the action directly to the notification body click.
ACTION=$(dunstify -u critical \
                  -t 8000 \
                  -A "default,Shut Down" \
                  "Power Button Pressed" "Tap notification to Confirm Shut Down.")

# Check if "default" or the numeric code for it was returned
if [ "$ACTION" = "default" ] || [ "$ACTION" = "2" ]; then
    poweroff
fi
[WHAT TO PUT INSIDE THAT power_dunst.sh END]
sudo chown root:root /usr/local/bin/dunst_power.sh

[Script that can be called with simply typing 'create_account.sh' in the Sakura or any other Terminal to create new user accounts]
sudo nano /usr/local/bin/create_account.sh
[WHAT TO PUT IN THAT create_account.sh START]
#!/bin/bash

# Force the script to run with administrative privileges automatically
if [ "$EUID" -ne 0 ]; then
    echo "This script needs admin privileges to create accounts."
    echo "Please enter your password below to proceed."
    exec sudo "$0" "$@"
fi

clear
echo "=========================================="
echo "    Chromebook New User Account Creator   "
echo "=========================================="
echo ""

# 1. Get the username
while true; do
    read -p "Enter a name for the new account (lowercase letters only, no spaces): " USERNAME
    # Convert input to lowercase and remove spaces for safety
    USERNAME=$(echo "$USERNAME" | tr -d ' ' | tr 'A-Z' 'a-z')
    
    if [ -z "$USERNAME" ]; then
        echo "The account name cannot be blank. Please try again."
        echo ""
    elif id "$USERNAME" &>/dev/null; then
        echo "An account with the name '$USERNAME' already exists. Please choose a different name."
        echo ""
    else
        break
    fi
done

echo ""
echo "------------------------------------------"
echo "        Account Permission Level          "
echo "------------------------------------------"
echo "Should this user be an Administrator?"
echo ""
echo " [N] Standard User (Recommended for guests/kids)"
echo "     - CANNOT install or remove software."
echo "     - CANNOT alter core system settings."
echo "     - Helps save storage space on your 16GB internal storage."
echo ""
echo " [Y] Administrator"
echo "     - CAN install new applications."
echo "     - CAN update the system and change settings."
echo "     - Full control over the entire Chromebook."
echo "------------------------------------------"

# 2. Ask for administrative privileges
while true; do
    read -p "Make this user an administrator? (y/n): " CHOICE
    CHOICE=$(echo "$CHOICE" | tr 'A-Z' 'a-z') # Convert to lowercase
    
    if [[ "$CHOICE" == "y" || "$CHOICE" == "yes" ]]; then
        GROUPS="audio,video,wheel"
        IS_ADMIN=true
        break
    elif [[ "$CHOICE" == "n" || "$CHOICE" == "no" || -z "$CHOICE" ]]; then
        GROUPS="audio,video"
        IS_ADMIN=false
        break
    else
        echo "Invalid input. Please type 'y' for Yes or 'n' for No."
    fi
done

echo ""
echo "Creating account for '$USERNAME'..."

# 3. Create the user with the dynamically selected hardware and system groups
useradd -m -G "$GROUPS" "$USERNAME"

# Check if user creation was successful
if [ $? -eq 0 ]; then
    echo "Account tracking folder created successfully."
    echo ""
    
    # 4. Handle password assignment
    echo "------------------------------------------"
    echo "Now, let's set a password for $USERNAME."
    echo "Note: The letters will NOT show on screen as you type them for safety."
    echo "------------------------------------------"
    
    while true; do
        passwd "$USERNAME"
        if [ $? -eq 0 ]; then
            echo ""
            echo "Success! The password has been set."
            break
        else
            echo "Passwords did not match or were too weak. Let's try again."
            echo ""
        fi
    done
    
    echo ""
    echo "=========================================="
    echo " Done! The account '$USERNAME' is ready."
    if [ "$IS_ADMIN" = true ]; then
        echo " Privilege Level: Administrator (Can install apps)"
    else
        echo " Privilege Level: Standard User (Restricted access)"
    fi
    echo " They can now log in to the Chromebook."
    echo "=========================================="
else
    echo "An unexpected error occurred while creating the account."
fi
[WHAT TO PUT IN THAT create_account.sh END]

[Script that can be called with simply typing 'delete_user.sh' in the Sakura or any other Terminal to list all erasable accounts and delete them if wanted]
sudo nano /usr/local/bin/delete_user.sh
[WHAT TO PUT IN THAT delete_user.sh START]
#!/bin/bash

# Force the script to run with administrative privileges automatically
if [ "$EUID" -ne 0 ]; then
    echo "This script needs admin privileges to delete accounts."
    echo "Please enter your password below to proceed."
    exec sudo "$0" "$@"
fi

clear
echo "=========================================="
echo "    Chromebook User Account Remover       "
echo "=========================================="
echo ""

echo "Fetching a list of accounts that can be deleted..."
echo "Press Alt+C to cancel this process anytime..."
echo "Beware of haphazardly using Alt+C to suddenly cancel Shell Operations..."
echo "------------------------------------------"

# List all user accounts on the machine with a User ID (UID) of 1000 or greater.
# This cleanly skips system profiles (like root, bin, daemon, pipewire) for safety.
USER_LIST=$(awk -F: '$3 >= 1000 && $1 != "nobody" {print $1}' /etc/passwd)

if [ -z "$USER_LIST" ]; then
    echo "No deletable user accounts found on this Chromebook."
    echo "=========================================="
    exit 0
fi

# Print the active accounts so the user can easily see them
echo "Found the following accounts on this machine:"
echo ""
echo "$USER_LIST"
echo "------------------------------------------"
echo ""

# 1. Ask for the target account name
while true; do
    read -p "Type the EXACT name of the account you want to delete: " TARGET_USER
    TARGET_USER=$(echo "$TARGET_USER" | tr -d ' ' | tr 'A-Z' 'a-z') # clean input
    
    if [ -z "$TARGET_USER" ]; then
        echo "The account name cannot be blank."
        echo ""
    elif ! id "$TARGET_USER" &>/dev/null; then
        echo "The account '$TARGET_USER' does not exist. Please check the list above."
        echo ""
    elif [ "$TARGET_USER" = "$SUDO_USER" ] || [ "$TARGET_USER" = "$(whoami)" ]; then
        echo "⚠️ WARNING: You cannot delete the account you are currently logged into!"
        echo "Please log into a different account if you want to delete this one."
        echo ""
    else
        break
    fi
done

echo ""
echo "------------------------------------------"
echo "        Storage Cleanup Option            "
echo "------------------------------------------"
echo "Should we delete their files too?"
echo ""
echo " [N] Keep Files (Safest)"
echo "     - Deletes the login account."
echo "     - KEEPS their documents and pictures inside /home/$TARGET_USER."
echo ""
echo " [Y] Erase Everything (Recommended for your 16GB Internal Storage)"
echo "     - Deletes the login account."
echo "     - PERMANENTLY erases all files, folders, and history for this user."
echo "     - Instantly frees up disk space."
echo "------------------------------------------"

# 2. Ask whether to purge data
while true; do
    read -p "Permanently erase their files and folders? (y/n): " CLEAN_FILES
    CLEAN_FILES=$(echo "$CLEAN_FILES" | tr 'A-Z' 'a-z')
    
    if [[ "$CLEAN_FILES" == "y" || "$CLEAN_FILES" == "yes" ]]; then
        DEL_FLAGS="-r -f"
        PURGE_DESC="Account and all files successfully wiped."
        break
    elif [[ "$CLEAN_FILES" == "n" || "$CLEAN_FILES" == "no" || -z "$CLEAN_FILES" ]]; then
        DEL_FLAGS="-f"
        PURGE_DESC="Account deleted. Their files were safely kept inside /home/$TARGET_USER."
        break
    else
        echo "Invalid choice. Please type 'y' for Yes or 'n' for No."
    fi
done

echo ""
echo "Processing removal for '$TARGET_USER'..."

# Kill any lingering background tasks running under the target user so userdel doesn't lock up
killall -u "$TARGET_USER" &>/dev/null
sleep 1

# 3. Execute the deletion command
userdel $DEL_FLAGS "$TARGET_USER"

if [ $? -eq 0 ]; then
    echo ""
    echo "=========================================="
    echo " SUCCESS! $PURGE_DESC"
    echo "=========================================="
else
    echo "❌ Error: Could not finish removing the account."
    echo "Ensure the user is fully logged out before attempting to delete them."
fi
[WHAT TO PUT IN THAT delete_user.sh END]
